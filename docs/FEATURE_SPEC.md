# FEATURE_SPEC

## 1. 목적

- 현재 구현된 관리자 화면의 기능을 코드 기준으로 정의한다.
- 기능 ID(`FS-xxx`)로 관리한다.
- 기존 화면의 동작과 문구를 유지하는 것을 최우선 기준으로 한다.
- 화면 구조와 IA는 [`docs/IA_SPEC.md`](./IA_SPEC.md)를 따른다.

## 2. 공통 규칙

### 2.1 상태값

| 상태 | 의미 |
|---|---|
| `IDLE` | 초기 대기 |
| `LOADING` | 조회 또는 처리 중 |
| `SUCCESS` | 정상 완료 |
| `EMPTY` | 결과 없음 |
| `ERROR` | 조회 또는 처리 실패 |
| `ACTIVE` | 활성 상태 |
| `INACTIVE` | 비활성 상태 |
| `LOCKED` | 잠금 상태 |

### 2.2 공통 UI

- `AuthScreen` : 로그인 / OTP 화면
- `Sidebar` : 사이드바
- `TopHeader` : 상단 헤더
- `DashboardShell` : 로그인 후 관리자 화면
- `ListPanel` : 목록 패널
- `DetailFrame` : 상세 패널
- `SectionHeader` : 섹션 헤더
- `Pagination` : 페이지네이션
- `ToastStack` : 토스트 스택
- 모달은 `main.js` 내부에서 각 화면별로 직접 렌더링한다.

### 2.3 공통 정책

- 긴 목록은 내부 스크롤을 사용한다.
- 선택한 항목은 목록과 상세를 분리해서 보여준다.
- 모달은 취소 / 확인 / 닫기 패턴을 따른다.
- 실패 상태는 기존 입력값을 유지한다.

## 3. 기능 정의

### 3.1 로그인 / OTP

| FS ID | 기능 | 설명 | 입력값 | 결과 | 현재 구현 |
|---|---|---|---|---|---|
| FS-001 | 로그인 | 아이디 / 비밀번호로 로그인한다. | `userId`, `password`, `rememberId` | OTP 모달 진입 | `AuthScreen` |
| FS-002 | OTP 인증 | Google OTP 등 운영 정책에 맞는 6자리 OTP를 입력한다. | `otp` | 대시보드 진입 | `AuthScreen` |
| FS-003 | 권한 차단 | 허용되지 않은 계정은 진입을 막는다. | `userId`, `password` | 안내 모달 | `AuthScreen` |
| FS-004 | OTP 실패 | OTP 실패를 누적 기록한다. | `otp` | 실패 횟수 표시 | `AuthScreen` |
| FS-005 | OTP 잠금 | 5회 실패 시 잠금 처리한다. | 없음 | 잠금 안내 | sessionStorage |
| FS-006 | 인증 유지 | 인증 상태를 유지한다. | 없음 | 재진입 가능 | session/localStorage |
| FS-007 | 로그아웃 | 인증을 해제하고 로그인으로 돌아간다. | 없음 | 인증 해제 | `Sidebar` / sessionStorage |

#### 정책

- 로그인 성공 후에만 OTP 모달을 연다.
- 구글 OTP는 운영 정책에 따른 값을 사용한다.
- 실패 시 안내 모달 또는 토스트를 사용한다.
- 잠금 상태는 로컬 저장소에 유지한다.

## 6. 비고

- 현재 구현은 `index.html`, `main.js`, `styles.css`로 구성된 단일 정적 배포 구조다.
- 별도 API 파일이나 분리 컴포넌트 파일은 없고, 화면별 로직은 `main.js` 내부 함수와 상태로 흡수돼 있다.
- 문서와 코드가 다르면 코드 기준을 우선한다.
- 구현 범위를 벗어나는 기능은 별도 문서로 분리한다.
