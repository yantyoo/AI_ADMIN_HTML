# Guidelines

## Product Rules

- 현재 구조는 `Vite + React + TypeScript SPA`다.
- 브라우저 URL 라우터보다 `App` 상태와 `DashboardShell` 내부 전환이 우선이다.
- 인증 흐름은 로그인 화면과 OTP 모달, 로그아웃 모달로 구성한다.
- 공통 패턴은 `ListPanel`, `DetailFrame`, `ModalDialog`, `ToastStack`을 유지한다.
- 로딩, 빈 상태, 오류, 비활성 상태는 각 화면별로 노출한다.

## Visual Rules

- 공유 테마 토큰을 유지한다.
- 기존 B2B 관리자 톤을 유지한다.
- 새로운 색 체계는 추가하지 않는다.
- 리스트/상세/모달 구조는 유지한다.

## Content Rules

- 문구는 짧고 운영 중심으로 작성한다.
- 마케팅성 문구는 사용하지 않는다.
- 버튼 라벨은 행동을 직접 설명한다.

## Structure Rules

| 구분 | 기준 |
|---|---|
| `app/*` | 화면 진입과 shell 연결 |
| `features/*` | 화면별 기능과 상태 관리 |
| `components/*` | 공통 UI/레이아웃 |
| `api/*` | mock 데이터와 로직 |
| `types/*` | 도메인 타입 정의 |

## State Rules

| 상태 | UI 기준 |
|---|---|
| `loading` | 버튼 비활성화, 로딩 문구 |
| `empty` | 결과 없음 안내 |
| `error` | 오류 안내와 재시도 경로 |
| `success` | 결과 표시와 후속 액션 노출 |
| `disabled` | 권한 없음 또는 입력 미완료 |

- 빈 상태는 오류가 아니다.
- 오류는 기존 입력을 유지한 채 보여준다.
- 검증 오류는 즉시 표시한다.

## Permission Rules

| 메뉴 | MASTER | OPERATOR |
|---|---:|---:|
| 대시보드 | O | O |
| 콘텐츠 관리 | O | X |
| 캐시 Q&A | O | X |
| 지식 기반 조회 | O | O |
| 피드백 관리 | O | O |
| 계정/권한 관리 | O | X |

- 메뉴 노출은 역할 기준으로 제한한다.
- 비노출 메뉴는 진입 자체를 막는다.
- 본인 계정은 권한 변경 대상에서 제외한다.

## Search / Pagination

- 검색은 화면 단위로 적용한다.
- 검색 결과는 정렬/필터 이후의 목록을 기준으로 보여준다.
- 페이지는 1부터 시작하고 총 페이지 수를 넘지 않게 clamp한다.
- 긴 목록 화면은 내부 스크롤을 사용한다.

## Error Policy

| 상황 | 정책 |
|---|---|
| 조회 실패 | 오류 상태와 재시도 버튼 표시 |
| 검증 실패 | 입력값 유지 + 필드 하단 문구 표시 |
| 중복 | 저장 전에 차단 |
| 권한 없음 | 버튼 비활성화 또는 화면 진입 차단 |
| 빈 결과 | 빈 상태와 다음 행동 안내 |

## Message Policy

| 유형 | 위치 | 기준 |
|---|---|---|
| 성공 | 화면 상단 또는 토스트 | 2.5~3초 자동 종료 |
| 실패 | 토스트 또는 모달 하단 | 기존 입력 유지 |
| 권한 없음 | 모달 또는 비활성 버튼 | 가능한 경우만 안내 |
| 복사 완료 | 토스트 | 즉시 피드백 |

## Notes

- 개발 산출물은 최신 구현을 기준으로 갱신한다.
- 문서와 코드가 다르면 코드를 우선한다.
- 미구현 기능은 `OPEN_QUESTIONS`로 분리한다.
